# Narrative-AI 종합 프로젝트 계획서
## AI 기반 맥락 중심 투자 시나리오 플랫폼

---

## 1. 프로젝트 개요

**프로젝트명**: Narrative-AI (맥락 기반 투자 시나리오 큐레이션 서비스)  
**버전**: 2.1 (현재 개발 중)  
**최종 업데이트**: 2026년 1월 30일  
**프로젝트 상태**: MVP 개발 완료, 배포 진행 중

### 1.1 비전 및 미션
> **"뉴스를 보지 말고, 흐름을 설계하세요"**

**핵심 가치**: 
- 투자자는 종목이 아닌 **투자 논리(시나리오)를 쇼핑**
- AI는 예언자가 아닌 **맥락 분석과 논리 구축을 돕는 참모** 역할
- 단순 차트 분석 대신 **인과관계 중심의 투자 접근법** 제공

### 1.2 문제 정의 및 해결 방안

| 문제점 | Narrative-AI 솔루션 |
|--------|-------------------|
| 정보 과부하로 중요한 투자 정보 파악 불가 | AI가 뉴스/공시 분석하여 핵심 시나리오만 선별 제공 |
| 유튜버/리딩방 의존으로 신뢰성 부족 | 투자 논리의 투명한 구조화 및 추적 가능성 |
| 단순 차트 분석만으로 차별화 불가 | 맥락 기반 시나리오 설계 및 논리적 투자 결정 |

## 2. 기술 아키텍처

### 2.1 시스템 구성도

```
Frontend (React/Vite) → API Gateway → 백엔드 서비스
    ↓
AI 시나리오 엔진 → 데이터 소스 (DART, 뉴스 API, KRX)
```

### 2.2 기술 스택 상세

**프론트엔드**:
- **프레임워크**: React 19.2.0 + TypeScript
- **빌드 도구**: Vite 7.2.4
- **상태 관리**: useState/useEffect (현재), 향상 Context/Redux 예정
- **스타일링**: CSS Modules + Framer Motion 애니메이션
- **PWA 지원**: Vite PWA 플러그인 통합

**백엔드 (개발 예정)**:
- **AI 통합**: OpenRouter API (GPT-4o-mini)
- **데이터 소스**: DART OpenAPI, 네이버 뉴스 API, KRX 수급 데이터
- **데이터베이스**: Vector DB (RAG 시스템용)

### 2.3 배포 환경
- **호스팅**: Vercel (현재 배포 완료)
- **도메인**: Custom Domain 설정 가능
- **CI/CD**: Git 기반 자동 배포

## 3. 핵심 기능 명세

### 3.1 시나리오 마켓 (Scenario Market) - ✅ 구현 완료

**기능 개요**: AI가 분석한 투자 테마별 시나리오 카드 제공

**주요 요소**:
- **Hero Section**: 오늘의 주요 시나리오 하이라이트
- **카테고리별 분류**: 안정형/공격형/배당형 시나리오
- **시나리오 카드**: [원인] → [현상] → [결과] 3단계 논리 흐름
- **데이터 필드**: 제목, 논리 흐름, 적중률, 관련 종목 Top 3

### 3.2 시나리오 빌더 (5-Step Wizard) - ✅ 구현 완료

**5단계 논리 설계 프로세스**:

| 단계 | 목표 | 핵심 질문 | 상태 |
|------|------|-----------|------|
| **1. 거시 테마** | 거시 경제 흐름 선택 | "지금 시장의 가장 큰 흐름은?" | ✅ |
| **2. 키 섹터** | 유망 산업군 선정 | "어떤 포인트에서 수익이 날까요?" | ✅ |
| **3. 주도 기업** | 섹터 내 1등 기업 선택 | "누구를 파티에 영입하시겠습니까?" | ✅ |
| **4. 벨류체인** | 핵심 파트너 선택 | "1등 기업과 함께 성장할 파트너는?" | ✅ |
| **5. 포트폴리오** | 최종 종목 결정 | "어떤 종목을 매수하시겠습니까?" | ✅ |

### 3.3 내러티브 대시보드 (My Narrative) - ✅ 구현 완료

**논리 상태 추적 시스템**:

| 상태 | 색상 | 의미 | 알림 |
|------|------|------|------|
| **ON_TRACK** | 🟢 초록 | 핵심 전제 유지 중 | - |
| **WARNING** | 🟡 노랑 | 전제 흔들림 (주의 필요) | 경고 알림 |
| **BROKEN** | 🔴 빨강 | 전제 붕괴 (매도 권장) | 즉시 알림 |

### 3.4 논리 기반 투자 인터페이스 - ✅ 구현 완료

**특징**:
- **"동의함" 버튼**: 계약서 서명 같은 무게감 있는 인터랙션
- **Exit Plan 설정**: 논리 훼손 시 자동 알림 시스템
- **시각화**: 텍스트 나열 대신 Flowchart 형태 논리 구조

## 4. 데이터 구조 및 AI 모델

### 4.1 데이터 모델

```json
{
  "scenario_id": "sc_20260130_001",
  "user_id": "user_123",
  "title": "HBM 낙수효과: 소부장 집중 공략",
  "logic_path": {
    "theme": "AI_SEMICON",
    "rationale": "HBM_SUPPLY_CHAIN", 
    "risk_factor": "NVDA_DROP_15PCT"
  },
  "portfolio": [
    { "ticker": "000660.KS", "name": "SK하이닉스", "weight": 0.5 }
  ],
  "status": {
    "logic_status": "ON_TRACK",
    "created_at": "2026-01-30T15:00:00Z"
  }
}
```

### 4.2 AI 통합 아키텍처

**챗봇 레이어 구조**:
- **챗봇 1 (Rule-based)**: 거시 테마 → 산업 선택 → 기업 Top 10 매핑
- **챗봇 2 (시각화)**: 이슈 선택 → 기업 분석 → 시각적 결과 제공

**데이터 소싱 전략**:
- **거시 테마**: 네이버 뉴스 API / BigKinds → LLM 요약
- **상승 논리**: DART OpenAPI (무료) 실시간 공시 데이터
- **수급 분석**: KRX 외국인/기관 순매수 데이터

## 7. 위험 관리 및 대응 계획

### 7.1 기술적 위험

| 위험 요소 | 가능성 | 영향도 | 대응 전략 |
|-----------|--------|--------|-----------|
| AI 모델 정확도 저하 | 중간 | 높음 | 다중 검증 + 인간 검수 시스템 |
| 실시간 데이터 지연 | 높음 | 중간 | 캐싱 시스템 + 폴백 데이터 |
| API 제한/비용 문제 | 중간 | 중간 | 다중 데이터 소스 백업 |

### 7.2 규제적 위험
- **투자 조언 금지**: AI는 정보 제공만, 투자 결정은 사용자 책임 명시
- **데이터 프라이버시**: 금융 데이터 보안 강화
- **법적 검토**: 금융 규제 기관 요구사항 준수

### 7.3 시장 위험
- **경쟁사 등장**: 빠른 기능 차별화 및 사용자 경험 최적화
- **사용자 수용도**: 직관적인 UI/UX와 교육 콘텐츠 제공

## 10. 결론 및 다음 단계

Narrative-AI는 기존의 단순 주식 추천 서비스를 넘어서 사용자가 **투자 논리를 이해하고 관리**할 수 있는 플랫폼으로 진화하고 있습니다. 현재 MVP 개발이 완료되어 실제 사용자 테스트를 진행할 준비가 되었으며, 지속적인 피드백을 통한 기능 개선이 예상됩니다.

**즉시 실행 가능한 다음 단계**:
1. 사용자 베타 테스트 진행 및 피드백 수집
2. 백엔드 API 개발 본격화
3. AI 모델 성능 최적화 작업
4. 추가 투자 유치 검토

이 문서는 Narrative-AI 프로젝트의 현재 상태와 향후 발전 방향을 종합적으로 제시하며, 지속적으로 업데이트될 예정입니다.
